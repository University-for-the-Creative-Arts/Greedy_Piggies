BP_ArchetypeComponent Documentation

Overview

Class Type: ActorComponent

Parent Class: ActorComponent

Description: BP_ArchetypeComponent is a modular cosmetic character identity system used by the player character in Greedy Piggies. Its purpose is to apply the correct archetype presentation data to a player at runtime without changing gameplay logic. This includes swapping the player mesh, returning the correct animation montages for gameplay events, and storing archetype-specific voice references. The component exists to keep the main Player Blueprint lightweight and to let designers edit archetypes using Data Assets rather than gameplay code.

Core Logic and Gameplay Systems

This component does not control gameplay mechanics such as card logic, scoring, movement, or authority. It only manages the player's cosmetic identity.

The system is built around four archetypes:

  • Dictator
  • Entrepreneur
  • Nepo Piggy
  • Nobody

Each archetype is represented by a Data Asset containing references to:

  • Character mesh
  • Card play montage
  • Card draw montage
  • Win montage
  • Lose montage
  • Audit montage
  • Shock montage
  • Win Voice
  • Lose Voice
  • Lose audit voice
  • Win audit voice
  • Shop access voice

At runtime, the player’s selected archetype is stored in BP_PlayerState. The Player Character reads this value and passes it to BP_ArchetypeComponent, which then loads and applies the corresponding Data Asset.

Logic Flow

Initialization: This component does not require heavy BeginPlay logic by itself. It is expected that the owning Player Character calls a refresh function, such as RefreshArchetypeFromPlayerState(), after PlayerState becomes valid. The component then receives an archetype enum and applies the relevant data.

Main Loop: This component is not designed around Tick-based behaviour. It is event-driven. It responds when:

  • The player initially spawns
  • The selected archetype is changed
  • The replicated PlayerState variable updates on clients

Dependencies:

  • BP_FirstPersonPlayer (or equivalent owning player character)
  • BP_Card
